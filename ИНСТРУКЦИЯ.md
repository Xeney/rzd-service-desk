# Инструкция по запуску RZD Service Desk

## Быстрый старт (локально)

### 1. Запуск через bat файл
Просто дважды кликни по файлу `start.bat` - откроются два окна:
- Backend: http://localhost:5000
- Frontend: http://localhost:5173

## Вариант 1: Vercel (бесплатно, самый простой)

### Подготовка Backend

1. Зарегистрируйся на https://vercel.com
2. Установи Vercel CLI: `npm i -g vercel`
3. В папке `backend` создай файл `vercel.json`:

```json
{
  "builds": [
    {
      "src": "app.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/app.py"
    }
  ]
}
```

4. Добавь в `backend/app.py`:
```python
import os
os.environ.setdefault('FLASK_ENV', 'production')
```

5. Запушь на Vercel: `vercel --prod`

### Подготовка Frontend

1. Собери проект: `npm run build`
2. Папка `dist` появится - это и есть готовый фронтенд
3. Загрузи эту папку на любой хостинг

### Деплой Frontend на Vercel

1. Зайди на https://vercel.com/dashboard
2. New Project → Import Git Repository
3. Выбери репозиторий с проектом
4. Framework Preset: Vue
5. Build Command: `npm run build`
6. Output Directory: `dist`
7. Deploy!

## Вариант 2: Railway (простой Python хостинг)

1. Зарегистрируйся на https://railway.app
2. Подключи GitHub репозиторий
3. Создай 2 сервиса:
   - Backend: `backend/app.py`
   - Frontend: собери (`npm run build`) и задеплой папку `dist`
4. В настройках укажи переменные окружения

## Вариант 3: Традиционный хостинг

### Backend (Python anywhere)

1. Зарегистрируйся на https://www.pythonanywhere.com
2. Открой Bash консоль
3. Клонируй репозиторий:
   ```
   git clone твой_репозиторий
   cd твой_проект
   ```
4. Установи зависимости:
   ```
   pip install -r backend/requirements.txt
   ```
5. WSGI файл:
   ```python
   import sys
   sys.path.insert(0, '/home/твой_логин/твой_проект')
   from app import app as application
   ```
6. Обнови настройки в Dashboard → Web → Add a new web app

### Frontend (любой хостинг)

1. Собери: `npm run build`
2. Загрузи папку `dist` на хостинг через FTP/SSH

## Вариант 4: Docker

### Dockerfile для Backend
```dockerfile
FROM python:3.10
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
EXPOSE 5000
CMD ["python", "app.py"]
```

### Сборка и запуск
```bash
docker build -t rzd-backend .
docker run -p 5000:5000 rzd-backend
```

## Простой бесплатный вариант

### Render.com

1. Зарегистрируйся на https://render.com
2. Создай Web Service для backend:
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python app.py`
   - Environment: Python 3.10
3. Создай Static Site для frontend:
   - Укажи папку `dist`
4. В frontend/src/api/index.js измени API URL на адрес backend

## Минимальный деплой за 5 минут

1. Зарегистрируйся на https://vercel.com
2. Импортируй проект
3. Vercel сам определит Vue и настроит
4. Получишь ссылку!
5. Для API нужно будет отдельное решение (например, PythonAnywhere для бэкенда)

---

После деплоя не забудь:
- Проверить CORS настройки в `backend/app.py`
- Поменять API URL во frontend если нужно
- Настроить базу данных (SQLite уже включён)
