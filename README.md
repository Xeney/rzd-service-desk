# RZD Service Desk

Корпоративная система Service Desk для ОАО «Российские железные дороги». Современное веб-приложение для эффективной обработки заявок сотрудников.

---

## Обзор

RZD Service Desk — это комплексная внутренняя система, предназначенная для упрощения управления заявками железнодорожников. Она предоставляет централизованную платформу для создания, отслеживания и решения различных типов запросов: оборудование, программное обеспечение и разрешения доступа.

### Ключевые функции

- **Многоролевой контроль доступа** — Сотрудник, Специалист и Администратор
- **Полный жизненный цикл заявки** — От создания до закрытия с полной историей изменений
- **Поддержка файлов** — Вложения и комментарии к заявкам
- **Адаптивный дизайн** — Работает на компьютерах и мобильных устройствах
- **Современный интерфейс** — Профессиональный UI с плавными анимациями

## Быстрый старт

### Требования

- Python 3.10 или выше
- Node.js 18 или выше
- Windows/macOS/Linux

### Установка

#### 1. Клонирование репозитория

```bash
git clone <адрес-репозитория>
cd rzd-service-desk
```

#### 2. Настройка Backend

```bash
cd backend

# Создание виртуального окружения (Windows)
python -m venv venv
venv\Scripts\activate

# Создание виртуального окружения (Linux/macOS)
python -m venv venv
source venv/bin/activate

# Установка ��ависимостей
pip install -r requirements.txt
```

#### 3. Настройка Frontend

```bash
cd frontend
npm install
```

### Запуск приложения

#### Вариант А: Скрипт быстрого запуска (Windows)

Просто дважды кликните по файлу `start.bat` — откроются backend и frontend.

#### Вариант Б: Ручной запуск

**Терминал 1 — Backend:**
```bash
cd backend
python app.py
```

**Терминал 2 — Frontend:**
```bash
cd frontend
npm run dev
```

## Роли и права доступа

### Сотрудник
- Создавать новые заявки
- Просматривать свои заявки
- Добавлять комментарии к своим заявкам
- Загружать файлы во вложения

### Специалист
- Все права Сотрудника
- Просматривать назначенные заявки
- Обновлять статус заявки
- Добавлять комментарии ко всем заявкам

### Администратор
- Все права Специалиста
- Управлять пользователями
- Управлять подразделениями
- Удалять заявки
- Полный доступ к системе

## API Reference

### Аутентификация

| Эндпоинт | Метод | Описание |
|----------|--------|----------|
| `/api/login` | POST | Аутентификация пользователя |
| `/api/logout` | POST | Завершение сессии |
| `/api/me` | GET | Информация о текущем пользователе |

### Заявки

| Эндпоинт | Метод | Описание |
|----------|--------|----------|
| `/api/tickets` | GET | Список заявок (с фильтрами) |
| `/api/tickets` | POST | Создание заявки |
| `/api/tickets/:id` | GET | Получение заявки |
| `/api/tickets/:id` | PUT | Обновление заявки |
| `/api/tickets/:id` | DELETE | Удаление заявки |

### Комментарии и вложения

| Эндпоинт | Метод | Описание |
|----------|--------|----------|
| `/api/tickets/:id/comments` | GET | Список комментариев |
| `/api/tickets/:id/comments` | POST | Добавление комментария |
| `/api/tickets/:id/attachments` | POST | Загрузка файла |

### Администрирование

| Эндпоинт | Метод | Описание |
|----------|--------|----------|
| `/api/users` | GET | Список пользователей |
| `/api/users` | POST | Создание пользователя |
| `/api/departments` | GET | Список подразделений |
| `/api/admin/departments` | POST | Создание подразделения |
| `/api/dashboard/stats` | GET | Системная статистика |

## Деплой

### Быстрый деплой (Vercel)

1. Загрузите код на GitHub
2. Импортируйте проект в Vercel
3. Настройте:
   - Framework Preset: Vue
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. Деплой!

### Деплой на PythonAnywhere

```bash
# Загрузите файлы backend
# Настройте WSGI
import sys
sys.path.insert(0, '/home/username/rzd-service-desk')
from app import application
```

### Docker

```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
EXPOSE 5000
CMD ["python", "app.py"]
```
